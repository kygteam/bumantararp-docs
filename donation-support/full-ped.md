Model **"Full Ped"** adalah aset karakter kustom penuh yang dirancang khusus untuk satu pemilik di kota Bumantara Roleplay.

Berbeda dengan karakter standar, model Full Ped menggunakan model unik dan **tidak dapat menggunakan pakaian** dari sistem *clothing system* server. Penampilan karakter bersifat permanen sesuai dengan ped custom yang dimasukkan.

Untuk memastikan kualitas dan optimisasi server, setiap pengajuan aset Full Ped **wajib** melalui proses peninjauan (vetting) oleh tim kami **sebelum** donasi dapat dilakukan.

### ğŸ¨ Kriteria & Standar
- **Kualitas Model:** Ped harus berkualitas baik dan dioptimalkan (low-poly) agar tidak mengganggu performa server atau menyebabkan *crash* pada pemain lain.
- **Lore-Friendly:** Desain ped harus realistis dan selaras dengan tema kota. Ped yang dianggap *fail roleplay* atau merusak imersi akan ditolak.
- **Bebas Bug:** Model tidak boleh memiliki *bug* visual yang parah (tekstur hilang, "bolong", dll.).
- **Evaluasi:** Tim Manajemen berhak penuh untuk menolak atau meminta revisi jika model ped tidak memenuhi standar.

### ğŸ”„ Prosedur Pengajuan dan Implementasi
Alur untuk menambahkan Full Ped adalah sebagai berikut:

#### **Langkah 1: Pengajuan & Peninjauan (Vetting)**
- **Pengajuan File:** Donatur mengajukan file ped kepada Tim Manajemen melalui tiket donasi untuk ditinjau.
- **Proses Peninjauan:** Tim akan memeriksa file untuk memastikan kesesuaian dengan semua Persyaratan Teknis dan Kriteria di atas.
- **Jika Ditolak:** Donatur akan diinformasikan mengenai masalahnya (misal: poly count terlalu tinggi) dan **wajib menyesuaikan filenya** terlebih dahulu.

#### **Langkah 2: Pembayaran Donasi**
- **HANYA SETELAH** file ped Anda dinyatakan **"Diterima"** oleh Tim Manajemen, Anda dapat melanjutkan ke tahap pembayaran donasi.
- **Biaya Bulanan:** Aset Ped Custom memiliki biaya perpanjangan bulanan untuk *maintenance*. Detail biaya akan diinformasikan saat konfirmasi.

#### **Langkah 3: Implementasi**
- **Aktivasi:** Setelah donasi dikonfirmasi, ped akan diimplementasikan ke server maksimal dalam **1x24 jam**.
